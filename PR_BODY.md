feat(chat): улучшенный фронтенд, переключение темы, пример прокси, обновлённые инструкции

Описание:

- Добавлен минимальный, современный фронтенд чата в `public/` (index.html, style.css, chat.js) — аккуратный UI, анимации, индикатор «печатает», плавный скролл, сохранение секрета в localStorage.
- Реализовано переключение темы Light / Dark с CSS-переменными, кнопкой и сохранением выбора в `localStorage`.
- Добавлен пример прокси `server/proxy.js` для вставки заголовка `Authorization` (опционально) и локальной отладки через Cloudflare Tunnel.
- Обновлены `package.json` (скрипты), `README.md` и `.github/instructions/copilot-instructions.md` (русская, проект-специфичная инструкция для агентов).

Тестирование (проверки для ревьюера):

- [ ] Запустить локальный dev-сервер: `npm run dev` и открыть <http://localhost:3000> — UI должен загрузиться.
- [ ] Проверить отправку сообщения: ввести текст и отправить; фронтенд должен POSTить на `https://n8n.mup.me/webhook/chat`.
- [ ] Если webhook защищён, установить секрет через кнопку «Установить секрет» и проверить, что заголовок `Authorization: Bearer <token>` отправляется.
- [ ] Проверить переключение темы (кнопка «Сменить тему»): выбор сохраняется после перезагрузки страницы.
- [ ] Проверить успешный ответ от n8n (примерный ответ: `{ "reply": "Пример ответа от n8n" }`).
- [ ] Код фронтенда: короткий ревью `public/chat.js` на обработку ошибок и XSS (сейчас используется эскейпирование).
- [ ] Прокси (опционально): запустить `node server/proxy.js` с `WEBHOOK_SECRET` в окружении и проверить, что запросы корректно форвардятся.

Файлы основных изменений:

- public/index.html, public/style.css, public/chat.js
- server/proxy.js
- package.json, README.md
- .github/instructions/copilot-instructions.md

Рекомендации по безопасности:

- Не хранить секреты в `public/`. Использовать прокси/сервер для секретов.
- Проверить, чтобы n8n не раскрывал чувствительные логи в ответах.

Примечания:

- Подробная инструкция по развертыванию добавлена в `DEPLOYMENT.md`.
